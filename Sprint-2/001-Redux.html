Managing application state is hard. We are going to use Redux to manage our application state.

Redux has a few concepts we need to learn.

State
The entire state of your application, represented as a single value (typically an object)
Reducer
Pure function that accepts state and an action, and returns a new state
Action
An "action" in Redux is simply an object with a "type" field, and optionally more fields.
Action Creator
Often we use a function to create an action, known as an Action Creator
